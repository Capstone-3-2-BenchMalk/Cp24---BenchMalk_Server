## Commit Convention

- feat: 새로운 기능 fix: 버그 수정
- design: ui 변경
- !breaking change: 커다란 api 변경
- !hotfix: 치명적인 버그의 급한 수정
- style: 코드 포맷 변경,세미 콜론 추가, 코드수정X
- refactor: 리팩토링
- comment:주석 추가 변경
- docs: 문서 수정
- chore: 빌드 테스트 업데이트, 패키지 매니저 수정
- rename: 파일명 변경
- remove: 파일 삭제 작업만 진행
